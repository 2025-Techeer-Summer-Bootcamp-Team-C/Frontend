# AI 가상 피팅 웹 서비스 - Claude 코드 사용 가이드

## 프로젝트 개요

이 프로젝트는 AI 가상 피팅 웹 서비스의 **피그마 디자인을 그대로 퍼블리싱**하는 프론트엔드 전용 프로젝트입니다.

### 이 프로젝트의 백엔드

django를 바탕으로 drf, REST API로 구현되어있음
경로: (Frontend 디렉토리 기준) ../Backend
API 연동이나 백엔드에서 데이터를 확인해야 한다면 읽기 전용으로 참고할 것.(절대 백엔드의 코드를 건드려선 안됨)

### 주요 특징

- 🎨 **피그마 디자인 퍼블리싱 전용**: 백엔드 로직 없이 순수 프론트엔드 구현
- 🚀 **모던 React + TypeScript**: 최신 기술 스택 활용
- 🎯 **함수형 컴포넌트 기반**: 모든 컴포넌트는 함수형 컴포넌트로 통일
- 📱 **PWA 반응형 디자인**: 웹에서 모바일까지 완벽한 반응형 지원
- 📦 **ES Module**: 최신 JavaScript 모듈 시스템 채택
- 🔧 **유연한 프로젝트 구조**: 컴포넌트 특성에 맞춰 구조 개조 가능

## 기술 스택

### Core Technologies

- **React 19.1.0** - 최신 React 버전 (함수형 컴포넌트 + Hooks)

- **Vite 7.0.3** - 빠른 개발 서버 및 빌드 도구
- **ES Module** - 최신 JavaScript 모듈 시스템

### Styling & UI

- **Tailwind CSS 4.1.11** - 유틸리티 퍼스트 CSS 프레임워크 (PWA 반응형 지원)
- **shadcn/ui** - Radix UI 기반 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리
- **React Icons** - 추가 아이콘 라이브러리
- **Icones** - 다양한 아이콘 라이브러리 통합 플랫폼

### Form & Validation

- **React Hook Form 7.60.0** - 폼 상태 관리
- **Zod 4.0.2** - 스키마 검증
- **@hookform/resolvers 5.1.1** - 폼 검증 통합

### Routing

- **React Router DOM 7.6.3** - 클라이언트 사이드 라우팅

### Development Tools

- **ESLint** - 코드 품질 관리
- **TypeScript ESLint** - TypeScript 린팅

## 프로젝트 구조

```
src/
├── components/          # 재사용 가능한 컴포넌트
│   ├── ui/             # shadcn/ui 기본 컴포넌트
│   ├── common/         # 공통 컴포넌트
│   ├── sections/       # 페이지 섹션 컴포넌트
│   ├── forms/          # 폼 관련 컴포넌트
│   └── dialogs/        # 다이얼로그 컴포넌트
├── pages/              # 페이지 컴포넌트
├── lib/                # 유틸리티 함수
├── assets/             # 정적 자산
├── hooks/              # 커스텀 훅
└── index.css           # 글로벌 스타일
```

> **참고**: 컴포넌트 특성과 개발 요구사항에 따라 프로젝트 구조를 유연하게 개조할 수 있습니다.

## Framelink Figma MCP 활용

### 기본 정보

- **Framelink Figma MCP**는 Figma 디자인 파일에 직접 접근하여 디자인 정보를 추출할 수 있는 Claude용 MCP(Model Context Protocol) 도구입니다.
- Figma API를 통해 디자인 토큰, 컴포넌트 정보, 레이아웃 구조, 스타일 정보 등을 자동으로 가져올 수 있습니다.
- Claude와 연동되어 디자인 파일을 분석하고 코드 생성에 필요한 정보를 제공합니다.

### 주요 기능

1. **디자인 토큰 추출**: 색상 팔레트, 타이포그래피, 간격, 그림자 등의 디자인 시스템 정보
2. **컴포넌트 분석**: Figma 컴포넌트의 구조, 속성, 스타일 정보 파악
3. **레이아웃 정보**: 정확한 크기, 위치, 간격, 정렬 정보 획득
4. **반응형 브레이크포인트**: 다양한 화면 크기별 디자인 정보
5. **이미지 및 아이콘**: 디자인에 사용된 이미지, 아이콘 자산 정보
6. **스타일 가이드**: 일관된 디자인 시스템 정보

### 활용 방법

1. **Figma 파일 접근**: Figma 파일 URL 또는 파일 ID를 통해 접근
2. **디자인 요소 분석**: 프레임, 컴포넌트, 스타일 정보 추출
3. **코드 생성 지원**: 추출된 정보를 바탕으로 정확한 코드 생성
4. **반복 개선**: 디자인 변경사항을 실시간으로 반영

### 권장 워크플로우

1. **Figma 디자인 파일 분석**: Framelink Figma MCP를 통한 상세 디자인 정보 추출
2. **외부 라이브러리 조사**: 디자인에 적합한 기존 컴포넌트가 있는지 우선 조사 후, 있으면 설치, 사용
3. **디자인 토큰 매핑**: 추출된 정보를 Tailwind CSS 설정에 적용
4. **컴포넌트 구조 설계**: 외부 라이브러리 없을 경우에만 커스텀 컴포넌트 설계
5. **반응형 구현**: 다양한 브레이크포인트별 레이아웃 구현
6. **스타일 적용**: 정확한 색상, 타이포그래피, 간격 적용

## 컴포넌트 라이브러리 활용 가이드

### 1. shadcn/ui (현재 설정됨)

- **설정**: `components.json`에 이미 설정되어 있음
- **사용법**: `npx shadcn@latest add [component-name]`
- **장점**: Radix UI 기반, 접근성 우수, 커스터마이징 용이

### 2. 추가 추천 라이브러리

#### Magic UI

```bash
yarn add @magic-ui/react
```

- **특징**: 애니메이션 중심, 인터랙티브 컴포넌트
- **적합한 용도**: 호버 효과, 로딩 애니메이션, 마이크로 인터랙션

#### Aceternity UI

```bash
yarn add framer-motion
```

- **특징**: Framer Motion 기반, 고급 애니메이션
- **적합한 용도**: 페이지 전환, 스크롤 애니메이션, 복잡한 인터랙션

#### Material UI (MUI)

```bash
yarn add @mui/material @emotion/react @emotion/styled
```

- **특징**: Google Material Design 기반, 완성도 높은 컴포넌트
- **적합한 용도**: 관리자 패널, 데이터 테이블, 복잡한 폼

### 라이브러리 선택 기준

1. **디자인 일관성**: Figma 디자인과 가장 유사한 스타일
2. **커스터마이징 용이성**: 디자인 요구사항에 맞게 수정 가능
3. **번들 크기**: 성능을 고려한 가벼운 라이브러리 선택
4. **접근성**: WCAG 가이드라인 준수

### 컴포넌트 개발 우선순위

1. **외부 라이브러리 조사**: 디자인에 적합한 기존 컴포넌트가 있는지 먼저 조사(magic mcp 사용)
2. **라이브러리 설치 및 적용**: 적합한 라이브러리가 있으면 즉시 설치하여 사용
3. **커스터마이징**: 외부 라이브러리 컴포넌트를 디자인에 맞게 수정
4. **커스텀 개발**: 적합한 외부 라이브러리가 없을 경우에만 새로 개발

## 아이콘 라이브러리 활용 가이드

### 1. Lucide React (현재 설정됨)

```bash
yarn add lucide-react
```

- **특징**: 깔끔하고 일관된 디자인의 아이콘 세트
- **사용법**: `import { IconName } from 'lucide-react'`
- **장점**: TypeScript 지원, 커스터마이징 용이, shadcn/ui와 호환성 우수

### 2. React Icons

```bash
yarn add react-icons
```

- **특징**: 다양한 아이콘 라이브러리 통합 (Feather, Heroicons, Material Icons 등)
- **사용법**: `import { FiHome } from 'react-icons/fi'` (Feather Icons)
- **장점**: 방대한 아이콘 컬렉션, 다양한 스타일 선택 가능

### 3. Icones

- **특징**: 다양한 아이콘 라이브러리를 통합한 검색 플랫폼
- **사용법**: [icones.js.org](https://icones.js.org)에서 아이콘 검색 후 해당 라이브러리 설치
- **장점**: 통합 검색, SVG 다운로드, 다양한 라이브러리 지원

### 아이콘 사용 원칙

1. **SVG 파일 직접 사용 금지**: 디자인에서 제공된 SVG 파일을 직접 import하지 않음
2. **라이브러리 아이콘 우선**: 동일한 모양의 아이콘을 라이브러리에서 찾아서 사용
3. **일관성 유지**: 프로젝트 내에서 동일한 아이콘 스타일 사용
4. **성능 최적화**: 필요한 아이콘만 import하여 번들 크기 최적화
5. **접근성 고려**: 아이콘에 적절한 aria-label 또는 title 속성 추가

### 아이콘 선택 워크플로우

1. **디자인 분석**: Figma에서 사용된 아이콘 식별
2. **라이브러리 검색**: Lucide React, React Icons, Icones에서 유사한 아이콘 검색
3. **최적 선택**: 디자인과 가장 유사하고 성능이 좋은 아이콘 선택
4. **일관성 확인**: 프로젝트 전체에서 사용되는 아이콘 스타일 통일
5. **접근성 적용**: 스크린 리더를 위한 적절한 속성 추가

## 개발 가이드라인

### 1. 컴포넌트 작성 원칙

- **외부 라이브러리 우선**: 기존 컴포넌트가 있는지 먼저 조사 후, 있으면 설치하여 사용
- **함수형 컴포넌트**: 모든 컴포넌트는 함수형 컴포넌트로 작성
- **단일 책임**: 하나의 컴포넌트는 하나의 역할만
- **재사용성**: props를 통한 유연한 커스터마이징
- **타입 안전성**: TypeScript 인터페이스 정의
- **접근성**: ARIA 속성 및 키보드 네비게이션 지원

### 2. 스타일링 가이드

- **Tailwind CSS 우선**: 유틸리티 클래스 활용
- **CSS 변수 활용**: 디자인 토큰을 CSS 변수로 관리
- **PWA 반응형 디자인**: 웹에서 모바일까지 완벽한 반응형 지원
- **다크모드 고려**: 색상 테마 시스템 구축

### 3. 성능 최적화

- **코드 스플리팅**: React.lazy() 활용
- **이미지 최적화**: WebP 포맷, lazy loading
- **아이콘 최적화**: 필요한 아이콘만 import, Tree shaking 활용
- **번들 최적화**: Tree shaking, 불필요한 의존성 제거

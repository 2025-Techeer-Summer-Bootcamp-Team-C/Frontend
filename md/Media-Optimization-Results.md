## 🔧 적용된 최적화 기법

### 1. 비디오 압축 설정
```bash
ffmpeg -i {input}.mp4 \
  -c:v libx264 \
  -crf 28 \
  -preset medium \
  -c:a aac \
  -b:a 96k \
  -movflags +faststart \
  {output}.mp4
```

**압축 파라미터 설명:**
- `libx264`: H.264 코덱 사용 (웹 호환성 최적화)
- `crf 28`: 품질 28 (23=고품질, 28=중품질, 범위: 0-51)
- `preset medium`: 압축 속도와 효율성 균형
- `b:a 96k`: 오디오 비트레이트 96kbps
- `faststart`: 웹 스트리밍 최적화 (moov atom을 파일 앞으로 이동)

### 2. 오디오 압축 설정
```bash
ffmpeg -i audio.mp3 \
  -codec:a libmp3lame \
  -b:a 96k \
  audio_optimized.mp3
```

**압축 파라미터:**
- `libmp3lame`: MP3 LAME 인코더 (고품질)
- `b:a 96k`: 오디오 비트레이트 96kbps (기존 128kbps에서 감소)

### 3. Lazy Loading 구현
- **Intersection Observer API** 활용
- 비디오가 화면에 보일 때만 로드
- 로딩 상태 표시로 UX 개선
- 페이드 인 애니메이션 적용

## ⚠️ 주의사항

### 품질 vs 용량 트레이드오프
- **CRF 28** 설정은 품질과 용량의 균형점
- 더 강한 압축 시 품질 저하 가능성
- 타겟 사용자층의 네트워크 환경 고려 필요

### 브라우저 호환성
- **H.264**: 모든 브라우저 지원 ✅
- **MP3**: 모든 브라우저 지원 ✅
- **Lazy Loading**: IE 지원 필요 시 polyfill 고려
